# FlowFilm v0.2 编辑器基础层设计

## 目标

冻结当前技术原型的功能扩张，集中完成一个可长期承载后续素材、剧情和 UI 系统的编辑器基础层。完成标准以整套画布工作流可连续使用为准，而不是单个控件存在。

## 画布状态架构

- React Flow 使用 `useNodesState` 与 `useEdgesState` 管理交互中的本地状态。
- 节点拖动过程中即时应用位置变化，保证节点持续跟随鼠标。
- 拖动结束后批量把最终坐标写回项目数据，触发保存和后续导出。
- 项目数据变化时同步节点内容和新增/删除节点，但不覆盖正在进行的拖动状态。
- 选择状态由 React Flow 维护，支持零个、一个或多个节点；属性面板显示最后一个主选中节点。

## 选择与导航

- 左键拖动画布空白区域执行框选。
- Shift 支持追加选择，框选和多选结果使用统一选择状态。
- 单击空白处清除节点与连线选择，属性面板显示空状态。
- 鼠标中键拖动或按住空格加左键拖动画布；滚轮缩放。
- 选中节点只增加外层强调描边，不覆盖顶部节点类型色。
- Ctrl+A 全选节点，Escape 清除选择。
- Home 适应全部节点，F 聚焦选中节点；工具栏提供等价图标命令。
- 显示当前缩放比例，并提供 100% 重置、放大、缩小和适应视图。
- 框选方向不影响结果，拖出画布后重新进入仍能正确结束选择。
- 多选节点拖动时保持相对位置，主选中节点在最前层显示但不遮挡端口。

## 节点创建与编辑命令

- 移除永久占据左侧的节点库。
- 双击画布空白处，在鼠标位置打开可搜索节点菜单。
- 菜单按“叙事、逻辑”分组，支持键盘搜索、Escape 关闭和点击空白关闭。
- 创建节点时使用鼠标对应的画布坐标，并自动选中新节点。
- 支持 Delete/Backspace 删除、Ctrl+C 复制、Ctrl+V 粘贴。
- 支持 Ctrl+X 剪切、Ctrl+D 原地偏移复制，并提供右键上下文菜单入口。
- 粘贴节点相对原位置偏移，生成新节点 ID；选择节点的选项 ID 同步重新生成，避免连接端口冲突。
- 支持 Ctrl+Z 撤销和 Ctrl+Shift+Z/Ctrl+Y 重做项目级编辑操作。
- 双击节点标题或按 F2 可重命名节点，Enter 确认，Escape 取消。
- Ctrl+F 打开“查找节点”，按名称和类型定位已有节点，不与“创建节点”菜单混用。
- 右键空白、节点和连线显示各自适用的上下文菜单；菜单不显示无效命令。
- 输入框、搜索框和文本区域聚焦时，删除、复制、撤销等快捷键只作用于文本编辑。
- 节点支持锁定位置；锁定节点仍可选择和编辑属性，但不能被拖动或框选移动。

## 连线与小地图

- 保留端口拖线和贝塞尔曲线。
- 新连接替换同一源端口的旧连接。
- 连线可选择并用 Delete 删除。
- 已有连线的端点可以拖到其他兼容端口完成重新连接。
- 禁止连接到自身和明显无效端口，拖线时用颜色提示可连接与不可连接状态。
- 连接被替换、删除和重新连接都进入撤销历史。
- 提供可选的网格吸附开关；关闭后节点可自由定位，开启后使用稳定网格间距。
- 小地图明确接收节点尺寸、颜色和位置，显示全部节点及当前视口。
- 小地图支持点击/拖动导航，保持紧凑尺寸。

## 组织与可读性

- 支持添加注释框，注释框可包含并随同移动多个节点，不参与剧情运行。
- 支持连线转接点，用于整理长距离或交叉连线，不改变运行逻辑。
- 节点菜单、查找结果和属性面板统一显示节点类型色与类型名称。
- 空画布、无搜索结果、无素材和无项目变量都有明确空状态。
- 画布在至少 300 个节点规模下仍需保持可拖动和可缩放，避免每次指针移动写入完整项目。

## 预览窗口

- 试玩预览改为画布浮动窗口，标题栏可拖动。
- 提供收起、展开试玩和重置位置命令。
- 右下角拖拽手柄调整大小，保持 16:9 内容比例和最小/最大尺寸。
- 预览位置与尺寸保存在 `localStorage`。
- 预览交互区域隔离画布拖动、缩放和框选事件。
- 拖动和缩放不能把窗口完全移出画布；窗口在画布尺寸变化后自动夹回可见区域。
- 预览窗口获得焦点时，空格、方向键和 Delete 不触发画布命令。

## 编辑器面板

- 左侧永久节点菜单移除，画布获得更多空间。
- 顶部工具栏增加“素材”和“项目”命令。
- 素材面板作为可开合侧边抽屉，保留导入、删除，并增加搜索、类型筛选、缩略/类型状态和引用数量。
- 项目面板作为可开合侧边抽屉，提供项目名称、导入、导出、变量列表、项目统计和最近保存状态。
- 属性面板保留右侧可调整宽度结构；无选择时显示明确空状态。
- 抽屉支持 Escape 关闭、标题栏关闭按钮和宽度调整，且不会同时覆盖画布两侧。
- 所有仅图标命令提供工具提示和可访问名称；禁用命令显示明确禁用状态。

## 历史与持久化

- 撤销栈保存结构化项目快照，不保存临时视口和悬浮窗口状态。
- 连续节点拖动只在拖动结束时生成一个历史记录。
- 项目内容继续自动保存到 `localStorage`。
- 画布视口、面板开合、面板宽度、预览位置和预览尺寸独立保存。
- 自动保存显示“保存中、已保存、保存失败”状态；失败时不伪装成成功。
- 页面异常关闭后可从最近一次有效快照恢复；损坏快照不会覆盖最后一个可用项目。
- 导入项目、批量删除、粘贴和删除选项均作为单个可撤销操作。

## 基础安全与恢复

- 删除多个节点前不弹出阻塞确认框，但操作必须可撤销。
- 删除仍被其他节点引用的素材时显示影响范围，并在确认后统一清理引用。
- 导入无效项目时保留当前项目，不进行半完成替换。
- 浏览器刷新或关闭前，如果存在尚未写入的项目变更，先完成同步保存。

## 验收标准

1. 按住节点移动时，每次指针移动都可看到节点跟随。
2. 空白拖动可框选多个节点，Shift 可追加，空白单击可全部取消。
3. 多选节点可以一起拖动、复制和删除。
4. 双击空白打开节点菜单，创建节点的位置与鼠标位置一致。
5. Ctrl+C、Ctrl+V、Ctrl+Z、Ctrl+Y 和 Delete 在输入框聚焦时不会误触。
6. 节点顶部类型色在选中状态下仍然清楚可见。
7. 小地图显示所有节点和视口，并能用于导航。
8. 预览可以拖动、缩放、收起，刷新后布局恢复。
9. 素材和项目面板默认不永久占用画布空间，可随时打开和关闭。
10. 刷新页面后项目内容、节点位置和编辑器布局均能恢复。
11. 全选、剪切、复制、粘贴、重复、重命名、查找和聚焦选中节点可用。
12. 右键菜单根据空白、节点和连线显示正确命令，输入框快捷键不会误操作画布。
13. 连线可重新连接，无效连接有明确反馈，且相关操作可撤销。
14. 注释框和转接点可以整理复杂图，但不会改变剧情运行结果。
15. 300 个节点的测试项目中，拖动过程不写入持久化项目且交互保持连续。

## 测试策略

- 单元测试覆盖历史栈、复制 ID 重建、项目坐标持久化和面板状态。
- React Testing Library 覆盖菜单创建、选择清空、快捷键保护和抽屉切换。
- 浏览器级测试覆盖真实 pointer 拖动、框选、多选拖动、预览拖动缩放和小地图显示。
- 生产构建、单文件生成和 GitHub Pages 部署必须通过后才交付。

## 明确延后到后续里程碑

- 自动布局、节点对齐和分布工具。
- 子图、宏、函数图和可复用节点模板。
- 多人实时协作、评论和云端版本历史。
- 插件市场、自定义脚本节点和第三方扩展 API。
- 多项目标签页、跨项目复制和资源包管理。
- 性能分析器、远程调试和完整断点系统。

这些能力在成熟编辑器中常见，但不是修复当前基础交互和建立稳定编辑器地基的前置条件。
