# FlowFilm v0.2 封版优化设计

## 目标

完成浏览器原型 v0.2 的交互与视觉封版，使核心节点编辑流程可以稳定测试，并为下一阶段节点能力和桌面化建设提供可靠基础。本轮不扩张完整节点体系，不实现桌面级多窗口管理，也不重建本地素材库。

## 方案选择

采用“专业编辑器”方案：借鉴 macOS 的清晰层级、克制材质、紧凑工具栏和明确控件反馈，但保持剧情节点编辑器需要的信息密度。本轮交付深色主题，并用设计令牌预留浅色主题；不在 v0.2 同时完成双主题重绘。

## 画布与选择

- 普通框选替换当前节点选择。
- `Shift + 框选`在框选开始时保留已有选择，并在结束时合并本次命中的节点。
- 节点单击、`Shift + 单击`、框选、全选和空白取消选择共享同一选择状态。
- 浏览器级验证框选单个节点、多个节点、追加框选和空白取消选择，不能只依赖组件测试。
- 小地图默认开启，画布控制区提供图标开关和工具提示。
- 小地图开关状态保存在编辑器本地偏好中，不写入项目文件。

## 素材缩略图

- 素材库、场景节点和属性面板中的图片与视频缩略图统一等比完整展示。
- 使用 `object-fit: contain`，剩余区域使用中性背景，不通过裁切填满容器。
- 节点缩略图保持稳定尺寸，不因不同素材比例改变节点布局。
- 浏览器版继续使用现有素材数据结构；项目文件夹、本地路径、代理视频、缺失文件修复和批量扫描留给桌面阶段。

## 保存反馈

- 自动保存和手动保存调用同一保存入口，使用统一状态流：`dirty -> saving -> saved | error`。
- 点击保存后按钮立即显示“保存中”；成功后显示勾选和“已保存”，失败时显示明确错误状态。
- 状态栏同步显示当前保存状态和最近保存时间。
- 保存成功只在实际完成 `localStorage` 写入后显示，失败不能伪装成成功。
- 项目变化后重新回到未保存或保存中状态。

## 通用控件反馈

- 所有可点击按钮统一提供悬停、按下、键盘聚焦、选中和禁用状态。
- 按下状态使用轻微位移、亮度或边框变化，避免改变控件尺寸和引发布局跳动。
- 图标按钮继续使用 Lucide 图标，并为含义不明显的按钮提供工具提示或无障碍名称。
- 反馈通过主题令牌和共享 CSS 状态实现，不逐个按钮维护互相不一致的样式。

## 项目设置

顶部工具栏新增“项目设置”入口，使用与素材和项目面板一致的临时可调宽抽屉。本轮包含：

- 项目名称。
- 画面比例，默认 `16:9`。
- 输出宽度和高度，默认 `1920 x 1080`。
- 当前互动 UI 参数：强调色、对话框透明度和按钮圆角。
- 主题项显示当前深色主题，并预留后续浅色与跟随系统选项；未实现的选项必须清楚标记为后续提供，不能表现为可用控件。

项目数据结构新增画面设置时提供默认值和旧项目兼容迁移，现有项目无需手工修改即可打开。

## 面板与窗口边界

- 试玩预览继续作为可移动、可缩放、可折叠的浮动窗口。
- 素材、项目和项目设置继续使用停靠抽屉，属性面板继续使用可调宽停靠面板。
- 所有板块自由脱离、拖动、缩放和停靠属于桌面版统一窗口管理，不在浏览器 v0.2 重复建设。
- 时间线继续默认收起。

## 性能与数据边界

- 增加约 300 个节点的性能测试数据，检查初次渲染、缩放、拖动、框选和小地图显示。
- 频繁交互中的临时状态保留在 React Flow 或局部状态中，只在操作边界写回项目模型。
- 本轮不解决大型视频存储。浏览器 `localStorage` 与 Base64 素材只用于原型验证；桌面版必须改为项目目录和本地文件引用，并按剧情邻近节点预加载媒体。

## 验证

自动化测试覆盖：

- `Shift + 框选`追加而非替换现有选择。
- 小地图开关与偏好持久化。
- 手动保存的保存中、成功和失败状态。
- 项目设置默认值、修改和旧项目兼容。
- 缩略图使用完整展示样式。

浏览器验证覆盖：

- 页面正确加载且无框架错误遮罩或相关控制台错误。
- 节点实时跟随拖动。
- 单节点框选、多节点框选、`Shift + 框选`、空白取消选择。
- 小地图开关及重新加载后的状态。
- 图片和视频缩略图不被裁切。
- 保存按钮和其他按钮的悬停、按下、聚焦与禁用反馈。
- 项目设置修改后预览比例和项目数据同步更新。
- 300 节点测试项目仍能完成缩放、拖动和框选操作。

完成后运行完整测试、TypeScript 检查、生产构建和单文件构建，更新 `outputs/flowfilm-engine.html`，提交并推送 GitHub Pages。

## 明确延期

- 完整浅色主题与跟随系统主题。
- 扩充完整互动影游节点类型。
- 桌面级多窗口、面板脱离与停靠系统。
- 本地文件夹素材库、代理媒体、素材重定位和大型项目媒体缓存。
- Steam 打包与发布流程。
