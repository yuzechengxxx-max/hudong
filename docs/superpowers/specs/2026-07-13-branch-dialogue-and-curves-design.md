# FlowFilm 分支、对话框与曲线连线设计

## 目标

让 FlowFilm 更符合互动影游制作习惯：节点连线使用明显曲线，视频场景可关闭视觉小说式对话框，选择节点可自由管理分支数量。

## 节点连线

- React Flow 连线统一使用贝塞尔曲线。
- 普通连线保持低对比度，选中节点发出的连线使用强调色。
- 连接端口、直接拖线、删除连线和项目数据格式保持不变。

## 场景对话框

- `scene` 节点新增 `showDialogue` 布尔字段。
- 新建场景默认 `showDialogue: false`，适合互动影游。
- 旧项目缺少该字段时按 `true` 迁移，避免已有视觉小说项目打开后丢失对白表现。
- 属性面板提供“显示对话框”开关。
- 开启时显示角色名、对白和继续按钮。
- 关闭且存在视频素材时，视频播放结束自动进入 `next` 连接。
- 关闭且不存在可播放视频时，预览右下角显示紧凑的继续按钮，避免剧情卡住。
- 编辑器预览和导出的独立网页作品使用相同行为。

## 可变分支数量

- 新建选择节点仍默认创建两个选项。
- 属性面板支持添加选项、删除选项、上移和下移。
- 最少保留一个选项，不设置固定上限。
- 每个新增选项生成稳定且唯一的选项 ID，并自动出现在节点连接端口中。
- 删除选项时同步删除以该选项 ID 为 `sourcePort` 的连线。
- 排序只改变显示顺序，不改变选项 ID 或现有连接。
- 节点高度随选项数量增长，所有端口保持对齐。

## 运行与导出

- 运行时继续使用选项 ID 查找对应出口，因此动态增删和排序不需要改变运行时接口。
- 导出网页根据 `showDialogue` 决定渲染对白层或纯视频播放。
- 纯视频播放使用 `ended` 事件推进；编辑器预览不再让视频循环播放。
- 图片素材或无素材场景没有自然结束事件，因此显示紧凑继续控件。

## 验证

- Schema 测试覆盖旧场景数据迁移和新场景默认值。
- 编辑器测试覆盖对话框开关、添加/删除/排序选项和连线清理。
- 图画布测试验证边类型为贝塞尔曲线和动态端口数量。
- 预览与导出测试覆盖有对话框、纯视频自动推进、无视频手动继续。
- 运行全部 Vitest、TypeScript 检查和 Vite 生产构建，并重新生成单文件交付物。
