# FlowFilm 节点拖动与常驻预览调整

## 目标

修复节点无法拖动的问题，缩小节点画布小地图，并将试玩预览改为画布左上角常驻浮窗。

## 交互设计

- 节点主体任意空白位置均可按住拖动，符合零代码用户的直觉。
- 节点连接端口继续用于拖线，不触发节点拖动。
- 节点内部未来增加的输入框、按钮等操作控件使用 `nodrag` 隔离，避免编辑时误拖节点。
- 单击节点仍然选中节点并更新属性面板。
- 节点坐标在拖动结束后写回项目数据，保持现有保存与导出格式不变。

## 预览布局

- 实时试玩预览常驻节点画布左上角，宽度约 280px，保持 16:9。
- 预览浮窗位于画布 UI 层，不随画布平移或缩放。
- 预览中的继续、选项、重新试玩和放大试玩保持可交互。
- 画布工具按钮移动到预览下方，避免遮挡。
- 右侧栏恢复为常驻属性面板，不再需要属性/预览页签切换。

## 小地图

- 小地图固定为约 140×96px，保留平移和缩放能力。
- 小地图继续放在右下角，与左上角预览和左下角缩放控件分离。

## 验证

- 组件测试验证节点主体不再带有 `nodrag` 类。
- 测试验证常驻预览无需切换页签即可显示。
- 测试验证小地图使用缩小后的明确尺寸。
- 运行全部 Vitest、TypeScript 检查和 Vite 生产构建。
- 重新生成 `outputs/flowfilm-engine.html` 单文件交付物。
