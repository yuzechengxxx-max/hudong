# FlowFilm v0.3 玻璃工作台 UI 系统设计

## 目标

把当前浏览器原型从传统三栏网页编辑器重塑为现代跨平台创作工具。采用“系统级玻璃工作台”方向，借鉴 macOS 的空间层级、材质、精细阴影和紧凑反馈，但不照搬系统窗口按钮或平台专属外观。现有剧情编辑、保存、素材、预览和导出功能保持不变。

## 范围

本轮重建设计系统和编辑器外壳：顶部工具岛、左侧工作区导航、节点、属性检查器、素材与项目抽屉、浮动试玩预览、小地图、状态浮层和完整主题系统。

本轮不扩充节点类型，不实现面板自由停靠与脱离，不改变项目数据和运行时规则。桌面级多窗口系统留给 Tauri 阶段。

## 空间结构

- 节点画布占据主工作区，不再被厚重的整高侧栏包围。
- 顶部使用悬浮工具岛，包含保存、素材、项目、设置、项目检查、试玩和导出。
- 左侧使用窄图标导航栏切换剧情、素材、项目和设置工作区，不恢复常驻节点菜单。
- 试玩预览继续可拖动、缩放和折叠，并改为玻璃浮层。
- 属性检查器视觉上采用浮动玻璃面板；浏览器版仍锚定右侧并支持调宽，避免提前建设桌面停靠系统。
- 素材、项目和设置继续作为临时抽屉打开，统一使用玻璃面板外观。
- 小地图和状态信息使用轻量浮层，小地图保持可开关。
- 时间线继续默认收起，需要时从底部展开。

## 视觉语言

- 玻璃材质只用于工具岛、导航、抽屉、预览和检查器等浮层。
- 节点、输入控件和画布保持实色，确保文字清晰且长时间使用不疲劳。
- 不使用装饰光球、彩色渐变背景或大面积单色染色。
- 节点类型通过窄色条、端口颜色和图标识别，节点主体保持中性。
- 圆角最大 8px；工具控件使用 5-7px，避免卡片化和过度柔软。
- 阴影用于表达浮层高度，不作为装饰。
- 字号与层级匹配工具密度，不使用营销页式大标题。

## 主题系统

提供 `dark`、`light` 和 `system` 三种编辑器主题偏好。

- 深色主题使用接近中性黑灰的表面，避免整页偏蓝。
- 浅色主题使用柔和灰白层级，避免纯白大面积刺眼。
- 跟随系统通过 `prefers-color-scheme` 实时响应系统变化。
- 主题偏好保存在编辑器本地设置中，不写入互动影游项目文件。
- 系统监听不可用时，`system` 回退到深色主题。
- 两套主题的尺寸、布局和交互完全一致，只改变设计令牌。

## 组件边界

建立通用编辑器组件：

- `IconButton`：图标、工具提示、选中、禁用和状态反馈。
- `SegmentedControl`：主题和工作模式切换。
- `Toggle`：二元设置。
- `Field`：输入、选择、数字、颜色和范围控件的统一外壳。
- `GlassPanel`：工具岛、抽屉、检查器和浮动窗口的材质与层级。
- `PanelHeader`：标题、辅助状态和关闭/折叠命令。
- `StatusBadge`：保存、项目健康和运行状态。

编辑器外壳拆为顶部工具岛、工作区导航、状态浮层和检查器外壳，减少 `App.tsx` 的视觉职责。业务状态仍由现有 `App` 与项目模型持有，视觉组件只接收明确属性和回调。

## 交互反馈

- 所有按钮具有悬停、按下、键盘聚焦、选中、禁用和加载状态。
- 图标按钮提供可访问名称和工具提示。
- 保存状态继续显示保存中、已保存和保存失败，并同时反映在工具岛与状态浮层。
- 打开抽屉、切换主题和折叠浮层使用短时淡入与位移动画。
- 动画遵守 `prefers-reduced-motion`，禁用后不影响操作反馈。
- 任何反馈不得改变控件尺寸或引发布局跳动。

## 响应式

- 桌面宽屏显示完整工具文字和右侧检查器。
- 中等宽度隐藏次要文字，只保留图标与工具提示。
- 窄屏优先保证节点画布；检查器和抽屉以覆盖层打开。
- 工具岛、浮层和节点不得发生文本溢出、遮挡或无法点击。
- 本轮目标是浏览器桌面与平板宽度，不为手机提供完整工程制作体验。

- 工具提示、工作区切换和主要浮层开合。
- 保存、素材、设置和预览现有功能未回归。

浏览器验证覆盖：

- 深色与浅色桌面截图。

- 节点拖动、框选、连接、素材拖放、保存、主题切换和试玩可连续完成。
- 页面无框架错误遮罩和相关控制台错误。
