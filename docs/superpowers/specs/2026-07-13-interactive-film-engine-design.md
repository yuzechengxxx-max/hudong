# 互动影游可视化引擎设计规格

## 1. 产品定位

本项目是一款浏览器本地优先、面向零代码用户的互动影游创作引擎。首版同时支持真人分支影视与视觉小说，以真人互动影视作为默认项目模板。用户可通过节点画布、表单式属性编辑、演出时间线、实时预览和可视化 UI 编辑器完成作品，并导出为可部署的网页互动影游。

首版是可实际制作小型完整作品的本地 MVP，不包含账号、云协作、服务器转码和应用商店原生包发布。

## 2. 长期演进目标

首版采用平台无关的引擎核心，后续可包装为 Electron 或 Tauri 桌面应用，并接入本地文件系统、FFmpeg、云存储和在线发布服务。项目格式保持开放、版本化，可进一步开发 Unity、Unreal 等第三方引擎导出器，但不承诺第三方引擎工程的直接无损转换。

## 3. 产品工作区

编辑器采用专业工具型工作台布局：

- 顶部：项目状态、诊断、试玩、发布和全局命令。
- 左侧：节点库、素材库与项目章节结构。
- 中央：可缩放、平移、拖拽和连线的剧情节点画布。
- 右侧：选中节点或资源的无代码属性检查器。
- 底部：视频、字幕、声音、互动与转场的演出时间线。
- 中央可停靠区域：实时游戏预览，可弹出或全屏。

实时预览支持从开头或选中节点运行、播放控制、设备比例切换、变量与路径调试，以及编辑后的热更新。

## 4. 系统架构

系统分为以下独立层：

1. 编辑器 UI：节点画布、属性面板、时间线、素材库、UI 编辑器和实时预览。
2. 编辑器核心：命令系统、撤销重做、自动保存、快照、项目诊断与导入导出协调。
3. 引擎核心：剧情图、变量、条件、时间线调度、节点注册和项目迁移。
4. 播放器运行时：媒体播放、剧情推进、互动、存档、UI 状态和素材预加载。
5. 平台适配器：浏览器存储、文件选择、项目备份和网页发布；未来替换为桌面或云端适配器。

编辑器和导出播放器必须共享同一引擎核心。引擎核心不得直接调用浏览器、Electron 或云服务 API。

## 5. 编辑与数据流

所有用户修改均转换为命令。命令更新项目模型后，依次触发增量校验、实时预览热更新、撤销历史记录和延迟自动保存。导出使用不可变项目快照，不直接读取正在变化的编辑状态。

项目采用带 `schemaVersion` 的 JSON 数据，媒体作为独立资源，以稳定资源 ID 引用。导入旧项目时逐版本迁移；迁移失败不得覆盖原文件。

## 6. 节点体系

首版节点分为六类：

- 内容：视频场景、图片场景、对白、旁白。
- 互动：普通选择、限时选择、输入、热点点击。
- 逻辑：条件判断、随机分支、变量修改、等待。
- 演出：转场、音乐、音效、字幕、角色进出场。
- 结构：章节入口、跳转、子剧情、检查点。
- 结束：普通结局、特殊结局、返回章节。

视频或图片场景可内嵌字幕、声音、转场和选项出现时间。只有改变剧情流向或需要复用的逻辑才必须成为独立节点。

节点类型通过注册表定义其端口、属性 Schema、默认值、校验规则和运行行为，为后续自定义节点与插件 API 留出边界。

## 7. 零代码交互规则

- 普通模式不展示表达式、脚本或 JSON。
- 条件以自然句式配置，例如“当好感度大于等于 3”。
- 变量操作以自然句式配置，例如“将线索增加 1”。
- 端口同时使用文字、形状和颜色标识。
- 添加选择节点时自动创建两个选项和出口。
- 删除已连接节点前展示受影响路径。
- 错误和警告可直接定位到节点、资源或 UI 组件。
- 高级模式才开放复杂条件、调试详情、插件节点与脚本扩展。

## 8. 项目模型

项目至少包含：

- 元数据、项目设置和格式版本。
- 章节、剧情图、节点、端口和连接。
- 全局与章节变量、道具和角色关系。
- 角色、素材索引及资源元数据。
- 演出时间线和媒体引用。
- UI 主题、布局、组件和响应式规则。
- 存档、回退和章节检查点规则。
- 发布、加载、错误页和作品信息设置。

项目 ID、节点 ID、资源 ID 和组件 ID 必须稳定，不能依赖显示名称。

## 9. 自定义 UI

首版包含三层无代码 UI 定制能力：

1. 主题：字体、字号、颜色、按钮、透明度、间距和动效参数。
2. 布局编辑器：拖拽对话框、字幕、选择按钮、倒计时、状态组件、菜单、存读档、图片、Logo 和装饰层。
3. 界面状态：播放、对话、选择、暂停、存档、加载和结局界面。

组件支持位置、尺寸、锚点、层级、对齐、显示条件、动画和变量绑定。首版提供桌面横屏、手机横屏和手机竖屏三个响应式断点，并显示安全区域与文本溢出警告。

自定义 CSS 和代码组件不在普通首版界面中提供，后续通过高级模式和插件接口加入。

## 10. 播放器运行时

运行时执行确定性的剧情图推进、条件判断、变量变化、时间线调度和 UI 状态切换。进入节点前预加载当前素材及下一跳候选素材。加载延迟时显示作品自定义加载界面。

运行时支持鼠标、触控和键盘；尊重减少动态效果等系统辅助设置。媒体播放失败时提供重试与可配置错误页，不得无限停留在黑屏。

## 11. 存档

支持自动存档、手动存档、章节检查点和多存档槽。存档包含当前节点、媒体时间、变量、道具、访问路径、项目版本和游玩时长。作者可配置回退、回看和重复选择策略。

项目升级后优先按稳定 ID 恢复旧存档；无法完整迁移时给出明确提示并保留原存档数据。

## 12. 项目诊断与错误恢复

阻断发布的错误包括缺失必需素材、没有有效入口、必经节点无出口、项目格式损坏和运行时无法识别的节点。不可达节点、过大素材、未设置封面和缺少移动端布局属于警告。

编辑器提供自动保存、操作历史和定时快照。项目加载失败时进入只读恢复模式；素材丢失时保留引用并支持批量重链接；存储空间不足时停止写入并提示导出备份。

## 13. 网页导出

发布流程为：完整诊断、冻结快照、收集实际使用资源、生成独立播放器和项目数据、输出静态目录或 ZIP、启动本地发布预览。

导出作品不得包含编辑器代码。导出结果应可部署到任意静态托管服务。首版不负责自动上传服务器或生成原生安装包。

## 14. 首版技术方向

采用 React 与 TypeScript 构建编辑器，使用成熟节点画布库实现节点交互，IndexedDB 保存本地项目与资源元数据。引擎核心、播放器、项目 Schema、编辑器和平台适配器按独立包或清晰模块边界组织。

具体依赖与目录结构在实施计划中根据当前空工作区确定。

## 15. 测试与验收

单元测试覆盖剧情推进、条件计算、变量、节点执行、存档恢复和项目迁移。编辑器集成测试覆盖命令、撤销重做、节点连接、自动保存、素材重链接和 UI 数据绑定。

端到端主路径为：新建真人影视模板、导入素材、建立分支、设置变量与条件、自定义 UI、从节点试玩、项目诊断、导出网页、打开成品并完成至少两个不同结局。

验收还包括桌面横屏、手机横屏、手机竖屏、键盘操作、触摸操作、慢速加载、缺失素材、损坏项目与存储空间不足场景。

## 16. 首版边界

首版包含本地项目中心、节点编辑、素材管理、变量与条件、演出时间线、实时预览、UI 编辑、诊断、存档和网页导出。

首版不包含账号、多人协作、云端素材、服务器转码、在线作品托管、付费系统、应用商店打包、插件市场和第三方引擎直接导出。这些能力通过现有分层和开放项目格式为后续版本保留扩展空间。
