# FlowFilm 节点系统第一阶段设计规格

## 1. 目标与范围

本阶段把 FlowFilm 从六种基础节点扩展为能够制作常规真人互动影游的完整剧情逻辑闭环，同时建立统一节点定义机制，避免编辑器、运行器、诊断器和导出播放器对同一种节点产生不同理解。

本阶段包含：

- 统一节点注册表。
- 跳转、章节入口、随机分支、等待、音乐、音效和限时选择节点。
- 更完整的变量运算、比较方式和类型校验。
- 按分类展示并可搜索的节点创建菜单。
- 与新增节点对应的试玩运行、网页导出和项目诊断。
- 从项目格式版本 1 到版本 2 的无损迁移。
- 面向大型项目的索引化运行与线性复杂度诊断边界。

本阶段不包含：

- 角色立绘、旁白和复杂对话演出等视觉小说专用能力。
- 第三方节点插件 API。
- 桌面文件系统、FFmpeg、Steam 打包和云协作。
- 新一轮整体界面重做。

## 2. 设计原则

1. 节点能力必须真实贯通创建、编辑、运行、诊断和导出，不能只出现于菜单。
2. 已确认的画布交互、窗口布局和主题行为保持不变。
3. 节点定义集中维护，界面文案、分类、颜色、端口和默认数据不再散落复制。
4. 项目文件保持开放、可迁移的 JSON，不依赖浏览器或未来桌面外壳。
5. 先服务真人互动影游的常见工作流，暂不为插件化和视觉小说能力提前增加复杂度。

## 3. 节点注册表

新增 `NodeDefinition` 注册表。每种节点定义包含：

- `kind`：稳定、可持久化的节点类型标识。
- `label` 与 `description`：面向创作者的中文名称和简短说明。
- `category`：内容、互动、逻辑、演出、结构或结束。
- `color`：节点识别色。
- `runtimeMode`：`visible`、`automatic` 或 `waiting`。
- `create`：生成完整默认节点数据的方法。
- `outputs`：依据节点数据生成端口的函数。
- `searchTerms`：搜索别名。

注册表负责编辑器元数据和结构规则，不在其中直接放 React 组件或运行器副作用。具体属性表单仍使用类型收窄后的专用编辑控件，避免建立过早、难维护的通用表单 DSL。

## 4. 首批节点体系

### 4.1 已有节点

- 故事入口：项目唯一入口，自动进入下一节点。
- 视频场景：播放视频或图片，可选择是否显示对话框。
- 玩家选择：显示任意数量选项。
- 条件判断：根据变量比较结果走成立或不成立端口。
- 修改变量：修改运行时变量后自动继续。
- 故事结局：结束当前流程。

### 4.2 新增节点

- 章节入口（结构）：为大型项目提供可命名的逻辑锚点，自动进入下一节点；首批不承担独立子图或文件拆分。
- 跳转（结构）：引用目标章节入口并直接转移，不使用普通输出连线；目标缺失时为发布错误。
- 随机分支（逻辑）：包含 2 至 8 个可编辑分支及正整数权重。运行时使用可注入随机源选择分支，默认等权。
- 等待（逻辑）：暂停指定毫秒数并进入等待状态；试玩可点击“跳过等待”，自动化测试使用显式 `resume()`，不依赖真实计时器。
- 音乐控制（演出）：播放、停止或淡出一个音频素材，然后自动继续。运行器产生音频指令，播放器负责执行。
- 音效播放（演出）：播放一个音频素材，可配置音量，然后自动继续。运行器产生一次性音效指令。
- 限时选择（互动）：在普通选择基础上增加时限和超时分支。玩家选择时按对应端口继续，超时则走 `timeout` 端口。

## 5. 变量与条件

变量继续支持数字、文本和布尔三种类型。本阶段扩展如下：

- 数字操作：设为、增加、减少、乘以、除以。
- 文本操作：设为、追加。
- 布尔操作：设为、切换。
- 通用比较：等于、不等于。
- 数字比较：大于、大于等于、小于、小于等于。
- 文本比较：包含、不包含。

属性面板根据变量类型只显示合法操作和值输入。项目诊断会报告变量不存在、操作与变量类型不匹配、除数为零等问题。首批条件节点仍为单条件双分支，不引入嵌套 AND/OR 条件组；复杂判断可串联多个条件节点完成。

## 6. 运行时与数据流

运行器预先建立节点 ID、出边和章节入口索引，避免每次推进都扫描完整项目。节点分为：

- 显示型：场景、普通选择、限时选择、结局，运行到此处将状态交给播放器。
- 自动型：入口、章节入口、跳转、条件、变量、随机分支、音乐和音效，执行后立即继续。
- 等待型：等待节点，返回 `waiting` 状态，收到 `resume()` 后继续。

`RuntimeSnapshot` 新增结构化 `pendingInteraction` 和 `effects`。`effects` 是运行器产生、播放器消费的命令列表，至少支持音乐播放、音乐停止、音乐淡出和音效播放。播放器完成消费后确认清除，防止恢复快照时意外重复执行。

限时选择使用运行器的 `timeout()` 事件推进。浏览器预览和导出播放器负责倒计时；核心运行器不读取系统时间，使测试、存档恢复和未来桌面端保持确定性。

随机分支允许在 `createRuntime` 中注入随机函数。生产环境默认使用 `Math.random`，测试使用固定随机值，从而得到可复现结果。

## 7. 项目格式与迁移

项目格式升级为 `schemaVersion: 2`。加载流程先识别版本，再执行逐版本迁移，最后用当前 Zod 模式校验。

版本 1 到版本 2 的迁移规则：

- 保留全部节点 ID、位置、边、素材和 UI 设置。
- 将旧条件操作符映射到新操作符集合。
- 将旧变量修改操作映射到对应的类型化操作。
- 为新增项目字段补齐默认值。
- 迁移失败只报告错误，不覆盖原始导入文件和当前已打开项目。

导出项目始终写入当前版本。保存、导入和网页导出共用同一加载与验证入口。

## 8. 编辑器体验

双击空白画布打开节点菜单的既有行为保持不变。菜单按六类分组，并提供即时搜索：

- 搜索匹配节点名称、说明和别名。
- 键盘上下键移动高亮，Enter 创建，Escape 关闭。
- 创建位置仍是双击时的画布坐标，不改变缩放级别。
- 暂不加入收藏和最近使用，待节点数量进一步增加后再评估。

新增节点的属性面板使用面向创作者的字段名称、素材选择器、单位和合法范围。随机分支和限时选择支持增删选项；删除选项时同时移除对应失效连线。

## 9. 诊断与错误定位

诊断器在现有缺失入口、悬空连线、缺少输出和不可达节点基础上增加：

- 重复入口或重复章节 ID。
- 跳转目标不存在。
- 必需端口未连接。
- 引用变量不存在或类型不兼容。
- 引用素材不存在或素材类型错误。
- 随机权重非法。
- 等待时长或限时选择时限非法。

每个可定位问题携带 `nodeId`，状态栏的问题列表点击后选中并居中该节点。阻止运行或发布的问题为错误，其余为警告。

## 10. 大型项目边界

本阶段不加载或转码真实大型视频，但数据结构和运行路径必须避免明显的项目规模瓶颈：

- 运行器创建时建立 Map 索引，单步推进不扫描全部节点和边。
- 诊断使用邻接表一次遍历图结构。
- 注册表为静态模块，不随渲染重复构建。
- 节点菜单只渲染搜索结果和分类内容，不读取素材二进制。
- 自动节点连续执行保留 1,000 步保护并返回可解释错误。

后续桌面版素材索引、缩略图缓存、视频代理文件和分章节懒加载另行设计。

## 11. 测试与验收

每项行为按测试驱动方式实现。验收至少覆盖：

- 注册表包含所有节点且端口随动态选项正确变化。
- 版本 1 项目可无损迁移并通过版本 2 校验。
- 每种新增节点在运行器中走到正确目标。
- 固定随机源可稳定选择预期分支。
- 等待、限时选择和音频效果可通过显式运行器事件测试。
- 非法变量、素材、跳转和端口产生可定位诊断。
- 节点菜单可搜索、键盘创建且不改变画布缩放。
- 试玩和导出播放器对新增节点行为一致。
- 现有画布平移、缩放、框选、多选、连线、小地图、点阵和浮动面板回归测试继续通过。
- TypeScript、单元测试、生产构建和桌面浏览器视觉检查全部通过。

## 12. 完成标准

创作者能在不写代码的情况下，用现有场景节点和本阶段新增节点制作包含章节跳转、变量判断、随机路线、限时选择、等待、背景音乐、音效和多个结局的真人互动影游；同一项目在编辑器试玩与网页导出中得到一致结果，错误能在发布前被定位。
