# FlowFilm 大型项目制作与项目管理设计规格

## 1. 目标

本阶段把 FlowFilm 从单画布剧情编辑器扩展为能够组织和维护中大型互动影游项目的制作工具。重点不是继续增加节点类型，而是让章节、变量、诊断、保存和性能形成可靠工作流。

完成后，创作者可以把项目拆成独立章节画布，管理全局变量，从检查结果直接定位问题，并在误操作或异常关闭后恢复最近版本。

## 2. 范围

本阶段包含：

- 版本 3 项目格式和版本 2 自动迁移。
- 独立章节画布及章节管理。
- 完整全局变量管理器。
- 跨章节诊断定位。
- 防抖自动保存、手动保存和本地恢复点。
- 300、1000、3000 节点项目基准。
- 浏览器存储适配器边界，为桌面项目目录做准备。

本阶段不包含：

- 桌面本地文件夹、FFmpeg、视频代理文件和 Steam 打包。
- 多人协作和云同步。
- 每章独立物理文件。
- 新一轮整体 UI 重做。
- 更多内容或演出节点。

## 3. 项目格式 v3

项目新增：

```ts
interface ChapterDefinition {
  id: string;
  name: string;
  order: number;
  entryNodeId?: string;
}

interface ProjectV3 {
  schemaVersion: 3;
  chapters: ChapterDefinition[];
  defaultChapterId: string;
  nodes: Array<StoryNode & { chapterId: string }>;
}
```

章节 ID 和节点 ID 均为稳定标识，改名和排序不得改变引用。普通边只能连接同一章节内的节点；跨章节流程必须通过跳转节点引用目标章节。

版本 3 的跳转节点显式区分两种目标：

```ts
type JumpTarget =
  | { targetType: "chapter"; targetId: string }
  | { targetType: "anchor"; targetId: string };
```

`chapter` 目标引用章节定义，`anchor` 目标引用旧章节入口节点的稳定锚点 ID。新建跳转默认使用章节目标；锚点模式仅用于兼容旧项目和同章高级跳转。

项目至少有一个章节。默认章节不能被删除，除非先指定新的默认章节。每个节点必须属于一个存在的章节。

## 4. 版本迁移

版本 2 项目加载时自动创建：

- 章节 ID：`main-story`。
- 章节名称：`主剧情`。
- 默认章节：`main-story`。
- 所有旧节点的所属章节：`main-story`。
- 旧跳转节点从 `chapterId` 迁移为 `{ targetType: "anchor", targetId: chapterId }`。

旧章节入口节点继续保留为剧情锚点，不自动转换成章节定义，避免改变运行流程。迁移不改变节点 ID、位置、边、变量、素材和 UI 设置。

加载仍通过唯一 `loadProject` 入口执行逐版本迁移和最终校验。迁移失败不得覆盖当前项目或原始导入文件。

## 5. 独立章节画布

编辑器维护 `activeChapterId`，它属于编辑器会话状态，不写入发布运行数据。`StoryGraph` 只接收当前章节的节点和同章节边。

章节切换行为：

- 项目抽屉显示按 `order` 排序的章节列表。
- 点击章节切换中央画布并恢复该章节上次视口。
- 顶部路径显示 `项目名称 / 章节名称`。
- 新建节点自动归属当前章节。
- 复制节点仅复制到当前章节。
- 删除章节前显示节点数和受影响跳转数量。

章节操作支持新建、改名、排序、复制和安全删除。复制章节会复制其节点和内部边，并重建节点、边和动态端口 ID；指向其他章节的跳转保持原目标。

章节删除有两种合法路径：空章节直接删除；非空章节必须明确确认，同时删除所属节点、内部边和指向这些节点的无效边。指向被删章节的跳转节点保留并由诊断器报告，避免静默改写剧情。

## 6. 跨章节运行

运行器继续对完整项目建立全局节点和章节索引。章节仅影响编辑器渲染，不限制运行器跨章节推进。

章节跳转从目标章节定义解析 `entryNodeId`。若章节没有入口、入口节点不存在或不属于目标章节，运行器返回明确错误，诊断器提前报告发布阻断问题。锚点跳转继续按旧章节入口节点的锚点 ID 解析。

同章节边不能连接到其他章节节点。该规则由项目命令和诊断器共同约束。

## 7. 变量管理器

项目抽屉提供独立变量视图，支持：

- 新建数字、文本和布尔变量。
- 编辑名称、类型和初始值。
- 搜索和按类型筛选。
- 显示引用节点数量。
- 点击引用定位节点。
- 安全删除。

变量名称供创作者阅读，变量 ID 用于持久化引用。改名不影响节点。

类型修改前检查全部引用：

- 所有引用与新类型兼容时允许修改，并转换初始值。
- 存在不兼容条件或运算时阻止修改，并列出引用节点。

删除未引用变量可直接执行。删除已引用变量必须先选择替换变量；本阶段不提供“强制删除并留下坏引用”。替换变量必须类型相同，替换在一个可撤销事务中完成。

## 8. 诊断定位

每个节点问题携带 `nodeId` 和可推导的 `chapterId`。点击检查结果执行：

1. 关闭检查弹窗。
2. 切换到节点所属章节。
3. 选中该节点。
4. 调用画布定位命令居中节点，保持当前缩放在合理范围。
5. 属性面板显示该节点。

项目级问题（例如缺少默认章节）只打开对应项目设置，不尝试定位节点。

新增诊断包括缺失章节、节点所属章节无效、跨章节普通边、章节入口无效和指向已删除章节的跳转。

## 9. 保存与恢复

新增平台无关存储契约：

```ts
interface ProjectRepository {
  loadCurrent(): Promise<Project | null>;
  saveCurrent(project: Project): Promise<void>;
  createRecoveryPoint(project: Project, reason: "manual" | "interval" | "migration"): Promise<void>;
  listRecoveryPoints(): Promise<RecoveryPointSummary[]>;
  restoreRecoveryPoint(id: string): Promise<Project>;
}
```

浏览器实现初期仍可使用 Web Storage，但 UI 和编辑器不得直接调用 `localStorage` 保存项目正文。

保存策略：

- 编辑操作后标记 `dirty`。
- 停止编辑 800 毫秒后自动保存。
- 手动保存立即执行并创建恢复点。
- 每 5 分钟且项目发生变化时创建恢复点。
- 迁移旧项目成功后，在首次写入 v3 前创建迁移恢复点。
- 保留最近 10 个恢复点，手动恢复点优先保留。

保存失败时继续保留内存项目，显示明确错误，不把状态标为已保存。

## 10. 恢复界面

项目抽屉提供“版本恢复”入口，显示时间、原因、项目节点数和章节数。恢复前先为当前状态创建恢复点，恢复后项目进入 `dirty` 状态，用户可检查后手动保存。

恢复不自动覆盖外部导入文件。

## 11. 性能边界

测试夹具生成多章节项目：

- 300 节点：基础交互回归。
- 1000 节点：单章压力上限测试。
- 3000 节点：分布在至少 10 个章节，验证章节切换和全局诊断。

性能约束：

- 画布只创建当前章节的 React Flow 节点和边。
- 章节切换不得重建项目运行器或复制整个项目正文。
- 诊断使用一次性全局索引和邻接表。
- 变量引用统计使用索引，不在每行渲染时扫描全部节点。
- 保存序列化不在拖动每一帧执行，只在事务完成后触发。
- 生产包代码分包列入本阶段，至少把导出播放器和非首屏弹窗逻辑从编辑器主包拆出。

性能测试记录数据规模、渲染耗时和命令耗时。CI 使用宽松回归阈值，本地浏览器 QA 记录实际指标，避免依赖不稳定的绝对毫秒断言。

## 12. UI 布局

项目抽屉使用两个标签页：`章节` 和 `变量`。它保持现有浮动、可缩放方式，不新增常驻侧栏。

章节列表是紧凑行式结构，不使用卡片嵌套。每行显示章节名称、节点数和问题数，更多操作放入菜单。变量列表显示名称、类型、初始值和引用数。

画布、试玩窗口、属性栏、小地图、点阵和既定手势保持不变。

## 13. 测试与验收

必须覆盖：

- v2 项目无损迁移到 v3。
- 新建、改名、排序、复制和删除章节。
- 当前章节过滤节点与内部边。
- 新节点自动写入当前章节。
- 跳转节点跨章节运行。
- 变量编辑、兼容类型修改、替换引用和安全删除。
- 诊断点击自动切章、选中并居中节点。
- 自动保存防抖、手动恢复点、失败状态和恢复流程。
- 300、1000、3000 节点夹具。
- 现有画布手势、预览、导出和主题回归。
- TypeScript、全量测试、生产构建、单文件构建和桌面视觉 QA。

## 14. 完成标准

创作者可以在一个项目中建立多个独立章节，在章节之间跳转，完整管理变量，从项目检查直接定位任意章节的问题，并从最近恢复点找回项目。3000 节点多章节项目不会把全部节点同时渲染到画布，现有互动影游试玩和网页导出行为保持一致。
